\chapter{Merchant Payment API}

Merchant payment APIs are called by a merchant backend, not by the Merchant
Dashboard. They use HMAC headers and are scoped to the authenticated merchant.

\section{\apiid{PAY-01} Create Payment}

Method and path & \code{POST /v1/payments} \\
Actor and auth & Merchant backend with HMAC headers. \\
Key request fields & \code{order_id}, \code{amount}, \code{description}, exactly one of \code{ttl_seconds} or \code{expire_at}, optional \code{metadata}. \\
Key response fields & \code{transaction_id}, \code{order_id}, \code{merchant_id}, \code{qr_content}, \code{status}, \code{expire_at}. \\
Important business rules & Merchant must be \code{ACTIVE}; duplicate pending create-payment is idempotent only when the semantic request is identical; successful prior payment blocks a new attempt for the same order. \\
Important error cases & \code{MERCHANT_NOT_ACTIVE}, \code{PAYMENT_PENDING_EXISTS}, \code{PAYMENT_ALREADY_SUCCESS}, validation errors. \\
Related scenarios & \code{PAY-01}, \code{PAY-04}, \code{PAY-05}, \code{PAY-06}, \code{PAY-07}, \code{PAY-08}, \code{PAY-10}. \\
\bottomrule
\end{longtable}
