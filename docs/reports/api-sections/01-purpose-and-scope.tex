\chapter{Purpose And Scope}

This document is the companion API documentation for the Mini Payment Gateway
report. The main report uses compact API family IDs when explaining the system
design. This document expands those IDs into endpoint contracts, shared request
and response conventions, error behavior, security notes, and Postman scenario
mapping.

The contract covers implemented features only:

\begin{itemize}
  \item merchant payment creation and lookup;
  \item provider or simulator callbacks for payment and refund outcomes;
  \item full refund creation and lookup;
  \item outbound merchant webhook delivery and manual retry;
  \item internal Ops APIs for auth, internal users, dashboard reads, merchant
  lifecycle, onboarding, credentials, portal-user provisioning,
  reconciliation, and audit-oriented recovery;
  \item Merchant Dashboard auth, overview, analytics, explorers, profile, and
  credentials.
\end{itemize}

Out-of-scope features are intentionally not documented as completed APIs:
settlement, disputes, export CSV, real-time polling, merchant self-service
onboarding, and partial refunds.

The Postman collections under \code{postman/} provide runnable scenario
evidence.
Scenario IDs such as \apiid{PAY-01}, \apiid{CB-01}, \apiid{REF-01}, and
\apiid{WH-10} are kept aligned with those collections where the scenario
documents already define IDs.
