\chapter{Shared Error Model}

All API errors use the same envelope shape:

\begin{lstlisting}[language=json,caption={Shared error response shape}]
{
  "error_code": "PAYMENT_ALREADY_SUCCESS",
  "message": "Payment already succeeded for this order.",
  "request_id": "req_...",
  "details": {}
}
\end{lstlisting}

\texttt{AUTH\_TIMESTAMP\_EXPIRED} & 401 & \code{X-Timestamp} is outside the validity window. \\
\texttt{AUTH\_INVALID\_SIGNATURE} & 401 & \code{X-Signature} does not match the canonical request signature. \\

\texttt{PAYMENT\_ALREADY\_SUCCESS} & 409 & The order already has a successful payment. \\
\texttt{REFUND\_WINDOW\_EXPIRED} & 409 & Refund is outside the seven day window from \code{paid_at}. \\
\texttt{REFUND\_AMOUNT\_NOT\_FULL} & 409 & Refund amount does not match the original payment amount. \\
\texttt{WEBHOOK\_RETRY\_NOT\_ALLOWED} & 409 & Webhook event is not eligible for retry. \\
\texttt{MERCHANT\_AUTH\_INACTIVE} & 403 & Merchant portal user is inactive. \\
