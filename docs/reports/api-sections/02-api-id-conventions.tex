The IDs are designed to be stable across:

\begin{itemize}
  \item the academic report;
  \item this companion API document;
  \item \code{docs/api/*.md};
  \item \code{docs/testing/scenarios/*.md};
  \item Postman scenario collections.
\end{itemize}

New IDs should only be introduced when the implemented API surface grows. They
should not be renumbered casually because the same IDs are reused in testing and
presentation material.
