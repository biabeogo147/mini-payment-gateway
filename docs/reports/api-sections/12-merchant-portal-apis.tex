\chapter{Merchant Portal APIs}

Merchant Portal APIs power the Merchant Dashboard. They are separate from
merchant HMAC APIs: dashboard login uses portal users, not merchant API
credentials.

Routes & \code{POST /v1/merchant-portal/auth/login}, \code{POST /v1/merchant-portal/auth/logout}, \code{GET /v1/merchant-portal/auth/me}, \code{POST /v1/merchant-portal/auth/change-password}. \\
Actor and auth & Merchant portal user with HttpOnly portal session cookie. \\
Key request fields & Login uses \code{merchant_id}, \code{email}, \code{password}; change-password uses \code{current_password} and \code{new_password}. \\
Key response fields & Current user identity, role, status, and \code{merchant_status}. \\
Important business rules & Invalid credentials return 401; inactive users return 403 with \code{MERCHANT_AUTH_INACTIVE}; password changes refresh the session and invalidate older credentials. \\
Important error cases & Login failure, inactive user, missing or invalid session, validation errors. \\
Related scenarios & Merchant Dashboard login, current-session, and password-change flows. \\

\midrule
\endhead
Method and path & \code{GET /v1/merchant-portal/analytics?days=7|30|90} \\
Actor and auth & Merchant portal session. \\
Key request fields & \code{days}, default 30. \\
Key response fields & \code{range}, \code{totals}, \code{series}, and \code{attention} objects with merchant-scoped analytics. \\
Important business rules & Only 7, 30, or 90 are accepted; zero-activity days are returned explicitly so charts remain stable; internal reconciliation state is not exposed to merchants. \\
Important error cases & Validation errors for invalid \code{days} values and session failures. \\
Related scenarios & Analytics page, merchant analytics tests. \\
\bottomrule
\end{longtable}
