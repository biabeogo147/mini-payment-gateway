# Product Docs

These files describe the intended product and business behavior. They are useful
when validating scope or deciding whether a new request belongs in the MVP.

- `scope.md` - product scope and out-of-scope boundaries.
- `business-flow.md` - business workflow narrative.
- `requirements.md` - functional requirements.
- `modules-and-entities.md` - module and core entity overview.
- `state-machine.md` - lifecycle state machines.
- `necessary-documents.md` - source document inventory from the original brief.
- `use-cases.md` - use-case description.
